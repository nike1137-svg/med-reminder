# PRD — 복약 관리 알리미 (MedCheck)

> 돌봄제공자가 어르신의 복약을 기록하고, 교대자·보호자와 현황을 공유하는 웹 서비스
>
> 작성일: 2026-07-22 · 작성자: 조경호

---

## 1. 문제 정의

어르신은 복용하는 약 종류가 많고(다제약물) 시간대별로 다르다. 인지저하가 있으면
**"약을 먹었는지 안 먹었는지" 자체를 기억하지 못해** 누락·중복 복용이 발생한다.

더 근본적으로, **인지저하·거동불편 어르신은 복용하고 싶어도 스스로 복용하지 못해**
복약이 돌봄제공자에게 의존하는 구조다. 그런데 요양보호사·가족이 교대할 때
인수인계가 **구두로만** 이뤄져 "아까 드셨나요?"가 반복되고,
멀리 있는 보호자는 **복용 현황을 확인할 방법이 없다.**

→ 이 서비스는 "어르신이 스스로 확인하는 도구"가 아니라
**"돌봄제공자가 '내가 드렸다'를 기록하고 교대자·보호자에게 넘기는 도구"**다.

## 2. 타겟 유저

- **주 사용자 — 돌봄제공자** (요양보호사·동거 가족): 약을 등록하고 복용을 체크한다.
- **보조 사용자 — 보호자** (떨어져 사는 자녀 등): 원격으로 복용 현황을 확인한다.
- 설계 원칙: 어르신 본인이 아니라 **돌봄제공자 중심**으로 설계한다.

## 3. 핵심 기능 (MVP)

1. **약 등록** — 약 이름 / 용량 / 복용 시간대 / 메모
2. **오늘의 복약 체크리스트** — 오늘 먹어야 할 약을 시간대별로 보여주고 체크
3. **복용 기록** — 언제 체크했는지 저장, 완료/미완료 구분
4. **약 목록 관리** — 등록된 약 조회·수정·삭제 (CRUD)
5. **미완료 강조** — 아직 안 먹은 약을 눈에 띄게 표시

### 시간대 처리 규칙

- 시간대 값: `아침` / `점심` / `저녁` / `취침` / `시간무관`
- 처방약: 정해진 시간대를 여러 개 지정 (예: 혈압약 = 아침, 저녁)
- 영양제·필요시 약: `시간무관` — **하루 1회, 언제 체크하든 완료 처리**
  (예: 영양제는 보통 아침이지만 점심에 드리기도 함 → memo에 "보통 아침" 기록)

## 4. 화면 구성 (2개 이상)

| 경로 | 화면 | 역할 |
| --- | --- | --- |
| `/` | 오늘의 복약 현황(홈) | 시간대별 오늘 체크리스트 + 미완료 강조 |
| `/medications` | 약 목록 | 등록된 약 전체 보기 + 수정/삭제 |
| `/medications/new` | 약 등록 폼 | 새 약 추가 |
| `/medications/[id]/edit` | 약 수정 폼 | 기존 약 수정 |

## 5. 데이터 구조

```
medications  (약 마스터 — 무엇을 먹어야 하는가)
  id            uuid (PK)
  name          text        예: "암로디핀 5mg"
  dosage        text        예: "1정"
  time_slots    text[]      예: ["아침","저녁"] / ["시간무관"]
  memo          text        예: "식후 30분 / 보통 아침"
  created_at    timestamptz

dose_logs    (복용 기록 — 실제로 먹었는가)
  id            uuid (PK)
  medication_id uuid (FK -> medications.id)
  date          date
  time_slot     text
  taken         boolean
  taken_at      timestamptz
  UNIQUE(medication_id, date, time_slot)
```

- 설계 원칙 1: **약(1) : 복용기록(N)** 1:N 관계로 분리한다.
- 설계 원칙 2: `time_slots`는 배열로 단순하게 잡고, 필요할 때 확장한다.
- 설계 원칙 3: **"오늘의 체크리스트"는 저장하지 않고** medications × 오늘 날짜로 매번 계산한다.
  `dose_logs`에는 체크된 것만 기록한다. (저장 데이터 vs 계산 데이터 구분)

## 6. API 명세

| 메서드 | 경로 | 하는 일 | CRUD |
| --- | --- | --- | --- |
| GET | `/api/medications` | 약 목록 조회 | Read |
| POST | `/api/medications` | 약 등록 | Create |
| PATCH | `/api/medications/[id]` | 약 수정 | Update |
| DELETE | `/api/medications/[id]` | 약 삭제 | Delete |
| GET | `/api/today` | 오늘 복약 현황(약 × 기록 조합) | Read |
| POST | `/api/doses` | 복용 체크/해제 (upsert) | Create/Update |
